Introduction

I use flex almost exclusively for the layout of web pages I create. Even so, I still need to remind myself of certain syntax when I am coding the HTML & CSS for a page. For my own easy reference I created this "reminder" page.

First came page layout using tables. It worked, but in a limited capacity and was terribly 'un-semantic'. Positioning too is limiting, even frustratingly so, when used for page layout (and IMHO impossible). Floating page elements can achieve complex layout patterns and arrangements with fairly good control - and is equally well supported by all browsers and devices - but it too has it's frustrating quirks. Now we have the CSS flexbox (flexible box) module. While it may not have 100% support across browsers (see caniuse for detail on support: IE seems to have the biggest issues), it will eventually and, being aware of those specific cases, flexbox is the best method for page layout.

This is by no means a comprehensive tutorial. My goal is to demonstrate some of the fundamental power of CSS flexbox. Hope it helps!

The Basics

A flex layout is composed of two types of elements: containers and items. This makes sense if you understand that "layout" is really a set of nested "boxes". There is the "parent" box (the container) and the "child" boxes contained within (the items). Like this:

We can manipulate the items within a container individually with flex whereas their container determines their overall layout. On a given page, we can define different containers whose items are laid out independent of one another much as we can (and have) with floating - but with much greater control.

Wrapping

Now we begin to get into flex properties that help to create more grid-like layouts. In this section we look at the flex-wrap property. It has 3 possible values:

  • no-wrap
  • wrap
  • wrap-reverse
